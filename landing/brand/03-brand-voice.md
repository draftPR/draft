# Alma Kanban - Brand Voice & Writing Samples

## Voice principles

### 1. Technical, not academic
We write for engineers who build things, not for people who read whitepapers. Use concrete language. Name specific tools, specific commands, specific outcomes.

### 2. Direct, not hyped
Say what the thing does. Don't say it's "revolutionary" or "game-changing." Engineers see through that. If it's good, the feature list speaks for itself.

### 3. Confident, not arrogant
We know what we built and why it matters. We don't need to trash competitors. We compare honestly and let the reader decide.

### 4. Concise, not sparse
Short sentences. But enough detail that the reader doesn't have to guess. "Three commands to get started" is better than "easy setup." Show the commands.

### 5. Builder ethos
We are engineers building for engineers. The tone is peer-to-peer, not vendor-to-customer. Think: explaining your project to a smart colleague at a coffee shop.

---

## Tone spectrum

| Dimension | Where we sit | NOT this |
|---|---|---|
| Formal <-> Casual | Casual-professional | Not corporate ("leverage synergies"), not bro-y ("this tool slaps") |
| Technical <-> Simple | Technical, assumes competence | Not dumbed-down ("click the button!"), not jargon-heavy for its own sake |
| Excited <-> Reserved | Measured confidence | Not breathless ("OMG this changes everything"), not dry/boring |
| Humble <-> Bold | Bold claims backed by specifics | Not vague hype, not self-deprecating |

## Writing samples

### Headlines

**Good:**
- "Stop babysitting your AI agents"
- "From goal to merged PR. No hand-holding."
- "The missing pipeline between 'describe what you want' and 'merge to main'"
- "AI agents write the code. You review the diffs. That's the whole workflow."

**Bad:**
- "Revolutionizing software development with AI" (vague, hyped)
- "The future of coding is here" (cliche, says nothing)
- "Supercharge your development workflow" (empty buzzword)
- "AI-powered project management platform" (sounds like a PM tool, not a delivery system)

### Subheadlines / descriptions

**Good:**
- "Describe a goal. Alma reads your codebase, generates scoped tickets, runs agents in isolated branches, verifies output, and delivers reviewed diffs."
- "Each ticket runs in its own git worktree. Agents never touch each other's work or your main branch."
- "Three commands. No account required."

**Bad:**
- "Our cutting-edge AI platform leverages advanced language models to streamline your development pipeline." (corporate speak, no specifics)
- "Build faster with the power of AI." (empty, could be any tool)
- "Alma Kanban uses state-of-the-art machine learning..." (academic, nobody cares about the ML)

### Feature descriptions

**Good:**
```
Git worktree isolation
Every ticket runs in its own branch. Agents never touch
each other's work or your main branch.
```

**Bad:**
```
Advanced Branch Management
Our sophisticated branching strategy ensures complete
isolation between concurrent workstreams, preventing
merge conflicts and maintaining code integrity.
```

**Good:**
```
Codebase-aware tickets
Tickets generated from your actual code, not from a prompt
that guesses at your project structure.
```

**Bad:**
```
Intelligent Ticket Generation
Leveraging AI to understand your codebase and automatically
create actionable development tasks with unprecedented accuracy.
```

### CTAs (Calls to Action)

**Good:**
- "Self-host now" (action + specificity)
- "Join waitlist" (honest about current state)
- "View on GitHub" (engineers' first instinct)
- "Get early access" (creates urgency without being pushy)

**Bad:**
- "Start your free trial" (there's no trial, it's self-hosted free forever)
- "Schedule a demo" (too enterprise, too salesy)
- "Unlock the future of development" (cringe)
- "Learn more" (lazy, says nothing)

### Social / short-form

**Good (X/Twitter voice):**
```
We built Alma Kanban because managing AI coding agents
shouldn't be a full-time job.

Describe a goal. Get scoped tickets. Agents run in isolated
branches. Tests run automatically. You review diffs.

Self-host free. No account needed.

github.com/doramirdor/alma-kanban
```

**Bad:**
```
Excited to announce Alma Kanban!!! The ULTIMATE AI-powered
development platform that will TRANSFORM how you build software!

Try it today and join the revolution!!!

#AI #DevTools #Startup #Innovation
```

## Words we use vs. words we avoid

| USE | AVOID |
|---|---|
| "goal" | "prompt", "instruction" |
| "ticket" | "task", "story", "work item" |
| "agent" | "AI assistant", "copilot" |
| "review diffs" | "quality assurance process" |
| "self-host" | "on-premise deployment" |
| "runs in isolation" | "sandboxed execution environment" |
| "verification" | "quality gate" |
| "ships" / "delivers" | "outputs", "generates artifacts" |
| "pipeline" | "workflow orchestration" |
| "git worktree" | "isolated branch namespace" |

## Formatting conventions

- **Monospace font** (IBM Plex Mono) for headings and product name
- Short paragraphs (2-3 sentences max)
- Bullet points over numbered lists (unless showing sequence)
- Code blocks for terminal commands (always show the actual commands)
- No emojis in product copy (except sparingly in social media)
- No exclamation marks in headlines or product descriptions
